# UX Polish Plan

상태 표기: [x] 구현됨 · [ ] 미구현

## High Impact (빠른 체감 개선)
- [ ] 파일리스트 키보드 내비게이션: ↑/↓ 이동, Enter 선택, Ctrl/Cmd+F 포커스
- [ ] 라벨 말줄임 + 전체값 툴팁: 과도한 길이 시 말줄임 처리, 호버 툴팁 제공
- [x] 라벨 순서 정돈: 폴더 → 파일명 → [필터] 순으로 표시(Analysis 뷰어 라벨)
- [ ] 정렬 옵션 강화: 이름/수정일 토글 지원(현재 이름 기준만)
- [ ] 안정 키 적용: `analysisFileSource`를 별칭 문자열 대신 FolderKey로 저장(표시는 별칭)
- [ ] 파일리스트 React key 안정화: `${source}-${file.name}-${file.lastModified}` 사용으로 깜빡임 방지

## Medium Impact (탐색/작업 흐름 개선)
- [ ] 폴더별 그룹 접기/펼치기: 큰 데이터셋에서 폴더 단위로 스캔 효율↑
- [ ] 썸네일 토글: 비동기 로드 + 캐시로 가독성 향상(옵션으로 성능 제어)
- [ ] 드롭 대상 하이라이트: 폴더 컨트롤에 드래그 오버 시 시각적 피드백
- [ ] 동일 파일명 충돌 처리 옵션: 덮어쓰기/건너뛰기/둘 다 유지 모달(현행 TEMP_n 자동 분산 유지)
- [ ] 필터 패널 그룹 접기/펼치기: 카테고리 탐색 피로 감소(상태 저장)
- [ ] 필터 파라미터 프리셋/최근 사용: 반복 작업 단축(최근 3개)
- [ ] 그리드 프리셋 버튼: 2x2/3x3/4x4 빠른 전환(미리보기 유지)

## Low Impact (완성도/접근성)
- [ ] ARIA 롤/포커스링: listbox/option 적용 및 키보드 포커싱 강화
- [ ] 한국어 UI 문구 통일: 텍스트 일관화(예: "All Folders" → "모든 폴더")
- [ ] 툴팁 문구 통일: 동사형, 짧고 일관된 톤(예: "뷰어 2 저장")
- [ ] 공통 파일리스트 아이템 컴포넌트 추출: Compare/Pinpoint/Analysis 재사용
- [ ] 선택 판별 유틸 추가: `getFileId(file, folderKey)` 및 동등 비교 헬퍼
- [ ] 타입 에러 정리: 미사용 변수/파라미터 제거로 ts-noise 감소
- [ ] 공통 라벨 컴포넌트화: 라벨 스타일/로직 일원화
