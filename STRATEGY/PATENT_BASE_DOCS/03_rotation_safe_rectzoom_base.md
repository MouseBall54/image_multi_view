# 03. 회전 안전(Rect-zoom) 역변환 줌 결정 로직 (바탕 문서)

> 고지: 본 문서는 기술/전략 참고자료이며 법률 자문이 아닙니다.  
> 기준 문서: [PATENT_PORTFOLIO_ANALYSIS_KR_PCT.md](../PATENT_PORTFOLIO_ANALYSIS_KR_PCT.md)

## 문서 목적
회전 상태에서 발생하는 사각 줌 오차 문제를 compareX가 어떻게 해결하는지 설명형으로 정리한다.  
목표는 "사용자 선택 영역과 실제 확대 결과를 일치시키는 원리"를 문서화하는 것이다.

## 기능 정의
이 기능은 사각 선택(Rect-zoom) 종료 시, 선택 중심점을 역회전/역스케일로 이미지 좌표에 투영한 뒤 다음 뷰포트 중심과 배율을 계산한다.  
Pinpoint(개별 줌)와 Compare/Analysis(전역 줌)에서 동일한 개념을 모드별로 적용한다.

핵심 포인트:
1. 2클릭 기반 사각 선택(시작/종료)
2. 선택 영역 중심의 역변환 좌표 계산
3. 선택 폭/높이 대비 화면 비율로 scaleFactor 산출

## 사용방법
대표 시나리오:
1. 이미지를 회전한 상태에서 Rect-zoom 모드를 켠다.
2. 첫 클릭으로 시작점을 지정하고, 마우스를 이동해 선택 사각형을 확인한다.
3. 두 번째 클릭으로 선택을 확정한다.
4. 확대 후 관심 영역이 화면 중심으로 정확히 들어오는지 확인한다.

운영 팁:
1. 극단적 회전(예: 90도 근처)에서도 동작을 비교해 회전 안전성을 확인한다.
2. Pinpoint/Compare 모드에서 같은 영역을 선택해 결과 일관성을 확인한다.

## 결과
정성 효과:
1. 회전된 이미지에서도 사용자가 지정한 영역을 예측 가능하게 확대할 수 있다.
2. 줌 후 재팬(pan) 반복 횟수가 줄어 작업 효율이 오른다.

관찰 포인트(측정 후보):
1. 선택 중심점과 확대 후 중심점 간 오차(px)
2. 목표 ROI 도달까지 필요한 추가 조작 횟수
3. 회전각 증가에 따른 줌 정확도 변화

## 강점
1. 회전/확대가 섞인 상황에서도 사용자 경험이 일관적이다.
2. 계산 절차가 명확해 테스트 자동화 포인트를 정의하기 쉽다.
3. 모드별 확대 방식(개별/전역)을 동일 개념으로 관리할 수 있다.
4. 미니맵/좌표 표시와 결합해 탐색 효율을 높일 수 있다.

## 특허 문서 전환을 위한 작성 포인트
권리화 핵심 구성요소:
1. 사각 영역 중심의 역회전/역배율 투영 단계
2. 선택 영역 대비 화면 비율 기반 차기 배율 산출 단계
3. 모드별 대상(개별/전역) 분기 처리

차별 주장 포인트:
1. 일반 Rect-zoom과 달리 회전 상태에서 의미 보존을 보장하는 계산 절차
2. 사용자 선택 의도와 결과 뷰포트의 정합성

실험/증빙으로 보강할 요소:
1. 회전각별 ROI 중심 오차 벤치마크
2. 기존 단순 줌 방식 대비 조작 횟수 비교

## 작성 시 주의사항/한계
1. "모든 해상도/비율에서 무오차"로 단정하지 않는다.
2. 입력 이벤트(2클릭) 의존성이 있으므로 UX 설명을 함께 유지해야 한다.
3. 특허 작성 시 제스처 일반론이 아닌 역변환 절차를 중심으로 한정해야 한다.
