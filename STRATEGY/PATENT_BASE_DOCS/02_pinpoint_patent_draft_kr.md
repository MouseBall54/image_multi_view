## 1. 문서 목적 및 법적 한계

### 쉬운 설명
이 문서는 compareX의 Pinpoint 기능을 "특허 출원에 바로 쓰기 쉬운 형태"로 정리한 문서다.  
핵심은 기술을 두 축으로 나눠서, 경쟁사가 일부만 흉내 내도 방어할 수 있게 준비하는 것이다.

### 출원용 기술 설명
본 문서는 Pinpoint 관련 기술을 다음의 2축으로 분리하여 권리화하는 초안을 제시한다.
1. A축: 창별 참조점 고정과 공통 화면 기준점 공유를 이용한 좌표 정합 엔진
2. B축: 레이아웃/배치/재배치/Pin-Pan/동시 탐색을 포함하는 사용자 주도 운영 시스템

본 문서에서 제시하는 청구항은 내부 검토용 초안이며, 법적 유효성 판단(FTO, 무효성 최종 감정)은 범위 밖으로 한다.

## 5. 배경기술 및 문제점

### 쉬운 설명
기존 도구는 확대하거나 회전하면 내가 보고 있던 위치가 자주 밀린다.  
그래서 다시 찾느라 시간이 오래 걸리고, 창이 많아질수록 더 헷갈린다.

### 출원용 기술 설명
기존의 중심 고정(center-based) 탐색 방식은 화면 중심을 기준으로 이동/회전/확대가 수행되므로, 사용자가 실제로 관찰하려는 관심 지점(결함/패턴/경계)이 지속적으로 유지되지 않는 문제가 있다.  
또한 다중 창 비교에서 창별 기준점이 일관되게 유지되지 않아 재팬(re-pan), 재줌(re-zoom), 재정렬(re-align) 조작이 반복되는 문제가 발생한다.

추가로 다음 문제가 확인되었다.
1. 레이아웃 확장 시(다수 창) 관찰 문맥이 쉽게 붕괴
2. 단일 이미지 내 복수 관심점을 병렬 검토하기 어려움
3. 자동 정합 기술이 있어도 사용자 확인 단계에서 좌표 일관성이 깨짐

## 8. A축 심층: 좌표 정합 엔진

### 8.1 핵심 개념

#### 쉬운 설명
각 창은 자기 기준점(refPoint)을 가지고, 모든 창은 같은 화면 기준점(refScreen)을 공유한다.  
이 두 점을 맞추는 계산을 반복하면, 확대/회전해도 비교 문맥이 무너지지 않는다.

#### 출원용 기술 설명
A축은 창별 이미지 좌표계와 공통 화면 좌표계를 분리 정의한다.  
창별 참조점과 화면 기준점을 동시에 만족하는 변환 파라미터를 산출하고, 해당 파라미터를 정방향 렌더링과 역변환 입력 해석에 공통 적용함으로써 폐루프 정합을 달성한다.

### 8.2 데이터 모델

| 변수 | 설명 | 구현 상태 |
| --- | --- | --- |
| `refPoint` | 각 뷰어 이미지 내 기준점(정규화 좌표) | [현재 구현] |
| `refScreenX`, `refScreenY` | 화면에서 기준점이 놓일 공통 위치 | [현재 구현] |
| `individualScale` | 뷰어별 확대 배율 | [현재 구현] |
| `globalScale` | Pinpoint 전역 확대 배율 | [현재 구현] |
| `totalAngleDeg` | 개별 회전 + 전역 회전 합 | [현재 구현] |
| `drawX`, `drawY` | 캔버스 배치 좌표 | [현재 구현] |
| `theta` | 라디안 회전각 | [현재 구현] |

### 8.3 정방향 변환 수식

#### 수식
```text
scale = individualScale * globalScale
theta = (totalAngleDeg * π) / 180

drawX = refScreenX - Sx - (cos(theta) * (ux - Sx) - sin(theta) * (uy - Sy))
drawY = refScreenY - Sy - (sin(theta) * (ux - Sx) + cos(theta) * (uy - Sy))
```

#### 쉬운 해석
"확대/회전된 이미지를 어디에 그려야 참조점이 목표 화면 위치에 딱 오느냐"를 계산한다.

#### 출원용 기술 설명
상기 정방향 수식은 창별 참조점의 회전/스케일 변환 후 좌표를 공통 화면 기준점에 일치시키도록 배치 파라미터(`drawX`,`drawY`)를 결정한다.

### 8.4 역변환 수식

#### 수식
```text
dx = screenX - centerX
dy = screenY - centerY

unrotX = centerX + dx * cos(-theta) - dy * sin(-theta)
unrotY = centerY + dx * sin(-theta) + dy * cos(-theta)

imgX = (unrotX - drawX) / scale
imgY = (unrotY - drawY) / scale
```

#### 쉬운 해석
사용자가 화면에서 클릭한 점을 다시 원본 이미지 좌표로 되돌린다.

#### 출원용 기술 설명
상기 역변환은 정방향 변환과 동일 파라미터(`theta`,`scale`,`drawX`,`drawY`)를 이용하므로, 표시 좌표와 입력 좌표 해석 간의 불일치를 줄이는 폐루프를 형성한다.

### 8.5 폐루프 정합 구조

#### 쉬운 설명
보이는 화면과 클릭 좌표가 "같은 규칙"으로 계산되기 때문에, 작업을 반복해도 기준이 쉽게 흔들리지 않는다.

#### 출원용 기술 설명
1. 정방향 변환으로 표시 좌표 생성  
2. 사용자 입력을 역변환으로 이미지 좌표 환산  
3. 환산 결과를 다시 참조점/조정 파라미터로 반영  
4. 동일 파라미터 집합으로 반복 운용  

상기 구조는 단일 방향 변환만 사용하는 구조 대비 정합 일관성을 높인다.

### 8.6 A축 기술효과

#### 쉬운 설명
같은 위치를 다시 찾는 시간이 줄고, 반복 작업에서 실수가 줄어든다.

#### 출원용 기술 설명
1. 기준점 드리프트 감소  
2. 회전/배율 조합 상황에서 좌표 해석 안정성 향상  
3. 반복 비교 시 재정렬 조작 횟수 감소  
4. 다중 창 비교의 정합 신뢰성 향상

### 8.7 A축 한계 및 경계

1. 자동 특징점 추출 기반 완전 자동 정합 기술이 아님  
2. 사용자 기준점 입력이 필요한 사용자 주도 정렬 기술임  
3. 좌표계(정규화/픽셀) 혼용 시 오차 해석이 왜곡될 수 있음

## 9. B축 심층: Pinpoint 운영 시스템

### 9.1 핵심 개념

#### 쉬운 설명
B축은 "실제로 사람이 일할 때 쓰는 순서" 자체를 기술로 만든 것이다.  
레이아웃을 정하고, 이미지를 창에 넣고, 핀 찍고, 함께 이동하며 비교한다.

#### 출원용 기술 설명
B축은 다중 뷰어 운영 과정을 단계화한다.
1. 레이아웃 선택(현재 UI 기준 최대 24 뷰어)
2. slot 단위 이미지 배치
3. Pin/Pan 입력 모드 분리
4. 창별 제어 + 전역 제어 혼합 운용
5. shared viewport 기반 동시 Pan
6. Shift/Swap 재배치 후 정합 문맥 유지

### 9.2 구성요소

| 구성요소 | 기능 | 구현 상태 |
| --- | --- | --- |
| Layout selector | rows x cols 선택, 24 뷰어 제한 | [현재 구현] |
| Slot 배치 | 드래그/드롭 및 자동 배치 | [현재 구현] |
| Pin 모드 | 창별 참조점 지정 | [현재 구현] |
| Pan 모드 | 공통 기준점 동시 이동 | [현재 구현] |
| 개별 회전/배율 | 창별 미세 조정 | [현재 구현] |
| 전역 회전/배율 | 전체 동기 조정 | [현재 구현] |
| Shift/Swap | 창 재배치 전략 | [현재 구현] |

### 9.3 운영 흐름

#### 쉬운 설명
실제 작업은 다음 6단계로 끝난다.
1. 창 배열 고르기
2. 이미지 배치
3. 창마다 핀 지정
4. 배율/회전 맞추기
5. 같이 Pan 이동
6. 필요하면 창 재배치

#### 출원용 기술 설명
상기 흐름은 단순 UI 나열이 아니라, 정합 문맥 유지라는 기술 효과를 가지는 연속 단계로 구성된다.  
특히 재배치 이후에도 기준점/변환 파라미터의 관계를 보존한다는 점이 핵심이다.

### 9.4 사용 시나리오

#### 시나리오 A: 다중 이미지 비교
1. 서로 다른 샘플 이미지를 다수 slot에 배치
2. 각 창에서 동일 유형 결함 지점에 핀 설정
3. 동시 Pan으로 넓은 영역을 한 번에 스캔
4. 전역 제어로 큰 틀 정합, 개별 제어로 미세 보정

#### 시나리오 B: 단일 이미지 다중 포인트 병렬 관찰
1. 동일 파일을 여러 slot에 중복 배치
2. 각 창마다 서로 다른 관심점에 핀 설정
3. 동시 Pan으로 공통 이동하면서 포인트 간 상대 관계 비교
4. 비교 전환 시간을 줄여 빠른 판독 수행

### 9.5 B축 기술효과

#### 쉬운 설명
창이 많아도 덜 헤매고, 같은 지점을 더 빨리 다시 찾을 수 있다.

#### 출원용 기술 설명
1. 재팬 횟수 감소  
2. 목표 지점 재도달 시간 단축  
3. 창 간 비교 전환 시간 단축  
4. 다수 뷰어(1/4/12/24) 환경에서 비교 문맥 유지

### 9.6 B축 한계 및 경계

1. B축 단독으로 자동 정합 정확도를 보장하지 않음  
2. 사용자 숙련도에 따라 초기 조작 성능 차이가 있을 수 있음  
3. 단순 동기 뷰어와 혼동되지 않도록 A축 결합을 명확히 기재해야 함

## 10. 대표 선행기술 대비 차이점

### 쉬운 설명
기존 특허는 자동 정렬 알고리즘 쪽이 많고, 우리 발명은 사람이 작업할 때 좌표와 화면을 안정적으로 연결하는 쪽에 강점이 있다.

### 출원용 기술 설명
아래 비교는 공개 웹 문헌을 기준으로 한 실무 스캔 결과이며, 본 발명은 A축/B축 결합 단위에서 차별성을 주장한다.

| 문헌 | 목적 | 구성 | 효과 | 본 발명 차이 |

선행기술이 자동 정렬 파라미터 추정에 집중하는 반면, A축은 사용자 기준점 고정과 입력-표시 일치 폐루프에 집중한다.

### B축 차이 요약
선행기술이 결과 표시 또는 알고리즘 처리 중심인 반면, B축은 레이아웃-배치-핀-조정-동시Pan-재배치를 하나의 시스템으로 정의한다.

## 12. 발명의 활용 및 침해판별 포인트

### 쉬운 설명
경쟁사가 비슷한 기능을 넣으면 화면 동작만 봐도 어느 정도 확인할 수 있다.

### 출원용 기술 설명
침해 가능성 판단은 소스코드 확보 없이도 UI 동작과 조작 패턴 분석으로 1차 스크리닝이 가능하다.

### 확인 체크리스트
1. 창별 기준점이 독립 저장되는가
2. 공통 화면 기준점으로 다중 창이 함께 이동하는가
3. 정방향 표시와 클릭 역변환이 같은 파라미터 집합을 공유하는가
4. 전역+개별 배율/회전을 혼합 운용하는가
5. 레이아웃 재배치 후 정합 문맥이 유지되는가
6. 단일 이미지를 다중 slot으로 병렬 검토할 수 있는가

### 경쟁사 대비 우수점
1. 자동정렬만 있는 도구 대비: 관찰 단계 반복 비용을 직접 줄임
2. 단순 동기 뷰어 대비: 기준점 고정으로 정밀 비교 신뢰도 향상
3. 소수 창 도구 대비: 최대 24 뷰어 운영으로 대량 비교 작업에 유리

## 13. 청구항 초안 (독립항 2개 + 종속항 12개)

> 아래 청구항은 내부 검토용 초안입니다.

### [청구항 1] 독립항 A1 (좌표 정합 엔진)
다중 뷰어 이미지 정합 방법에 있어서,  
(a) 각 뷰어별 이미지 기준 참조점(refPoint)을 저장하는 단계,  
(b) 상기 다중 뷰어에 공통으로 적용되는 화면 기준 좌표(refScreenX, refScreenY)를 저장하는 단계,  
(c) 뷰어별 개별 배율과 전역 배율을 결합하여 총배율을 산출하고, 뷰어별 개별 회전과 전역 회전을 결합하여 총회전각을 산출하는 단계,  
(d) 상기 총배율, 총회전각, 상기 참조점 및 상기 화면 기준 좌표에 기초하여 이미지 배치 좌표를 산출하는 정방향 변환 단계,  
(e) 사용자 입력 화면 좌표를 상기 정방향 변환과 동일 파라미터 집합에 기초하여 원본 이미지 좌표로 환산하는 역변환 단계  
를 포함하는 것을 특징으로 하는 다중 뷰어 이미지 정합 방법.

### [청구항 2] 종속항 A2
청구항 1에 있어서, 상기 참조점은 정규화 좌표계(0~1 범위)로 저장되는 것을 특징으로 하는 방법.

### [청구항 3] 종속항 A3
청구항 1에 있어서, 상기 정방향 변환은 참조점이 화면 기준 좌표에 일치하도록 drawX, drawY를 계산하는 것을 특징으로 하는 방법.

### [청구항 4] 종속항 A4
청구항 1에 있어서, 상기 역변환은 회전 중심 기준 역회전 연산 후 스케일 및 평행이동 역적용을 수행하는 것을 특징으로 하는 방법.

### [청구항 5] 종속항 A5
청구항 1에 있어서, 상기 총배율은 `individualScale * globalScale` 연산으로 산출되는 것을 특징으로 하는 방법.

### [청구항 6] 종속항 A6
청구항 1에 있어서, 상기 총회전각은 뷰어별 회전값과 전역 회전값의 합으로 산출되는 것을 특징으로 하는 방법.

### [청구항 7] 종속항 A7
청구항 1에 있어서, 상기 정방향 변환 결과와 역변환 결과의 반복 적용으로 기준점 드리프트를 검증하는 단계를 더 포함하는 것을 특징으로 하는 방법.

### [청구항 8] 독립항 B1 (Pinpoint 운영 시스템)
다중 뷰어 기반 사용자 주도 정밀 정합 시스템에 있어서,  
(a) 레이아웃 선택부,  
(b) 선택된 레이아웃의 slot 단위로 이미지를 배치하는 배치부,  
(c) Pin 입력 모드와 Pan 입력 모드를 분리 제공하는 입력 제어부,  
(d) 창별 배율/회전 제어와 전역 배율/회전 제어를 결합 제공하는 변환 제어부,  
(e) 공통 화면 기준 좌표를 갱신하여 다중 뷰어 동시 Pan을 수행하는 동기 이동부,  
(f) slot 재배치(shift 또는 swap) 후에도 정합 문맥을 유지하는 재배치부  
를 포함하는 것을 특징으로 하는 사용자 주도 정밀 정합 시스템.

### [청구항 9] 종속항 B2
청구항 8에 있어서, 상기 레이아웃 선택부는 최대 24개의 뷰어를 지원하도록 구성되는 것을 특징으로 하는 시스템.

### [청구항 10] 종속항 B3
청구항 8에 있어서, 상기 배치부는 동일 이미지 파일을 복수 slot에 동시 배치 가능하도록 구성되는 것을 특징으로 하는 시스템.

### [청구항 11] 종속항 B4
청구항 8에 있어서, 상기 입력 제어부는 Pin 모드에서 참조점 설정을 수행하고 Pan 모드에서 공통 화면 기준 좌표 이동을 수행하도록 구성되는 것을 특징으로 하는 시스템.

### [청구항 12] 종속항 B5
청구항 8에 있어서, 상기 재배치부는 shift 모드 및 swap 모드를 선택적으로 수행하도록 구성되는 것을 특징으로 하는 시스템.

### [청구항 13] 종속항 B6
청구항 8에 있어서, 상기 시스템은 다중 이미지 비교 시나리오 및 단일 이미지 다중 관심점 병렬 관찰 시나리오를 선택적으로 수행하는 것을 특징으로 하는 시스템.

### [청구항 14] 종속항 B7
청구항 8에 있어서, 상기 변환 제어부 및 동기 이동부는 청구항 1의 좌표 정합 방법에 따른 변환 파라미터를 참조하여 동작하는 것을 특징으로 하는 시스템.
