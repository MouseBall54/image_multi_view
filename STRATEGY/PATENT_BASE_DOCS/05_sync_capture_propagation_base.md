# 05. 모드별 필터 상태 동기 전파(sync-capture) (바탕 문서)

> 고지: 본 문서는 기술/전략 참고자료이며 법률 자문이 아닙니다.  
> 기준 문서: [PATENT_PORTFOLIO_ANALYSIS_KR_PCT.md](../PATENT_PORTFOLIO_ANALYSIS_KR_PCT.md)

## 문서 목적
기준 뷰어의 필터 상태를 다중 뷰어에 빠르게 복제하는 sync-capture 기능을 설명형으로 정리한다.  
핵심은 "모드별 키 구조 차이를 어떻게 숨기고 일괄 전파하는가"다.

## 기능 정의
sync-capture는 사용자가 기준 뷰어를 선택하면 해당 필터 타입과 파라미터를 현재 모드의 활성 뷰어 집합에 일괄 적용하는 기능이다.  
Analysis 모드(숫자 인덱스 키)와 Compare/Pinpoint 모드(폴더 문자 키)를 각각 처리한다.

핵심 포인트:
1. 동기화 상태(`active`, `mode`) 관리
2. 기준 뷰어 필터/파라미터 추출
3. 활성 뷰어 집합에 모드별 키 매핑 후 일괄 반영

## 사용방법
대표 시나리오:
1. 모드에서 기준 뷰어 하나에 원하는 필터와 파라미터를 설정한다.
2. Sync 버튼을 활성화한다.
3. 기준 뷰어를 클릭해 동기화 소스로 지정한다.
4. 다른 활성 뷰어들에 동일 필터 상태가 반영되는지 확인한다.
5. 필요 시 다시 기준 뷰어를 바꿔 재적용한다.

운영 팁:
1. 동기화 전 기준 뷰어 파라미터가 확정되었는지 먼저 점검한다.
2. 레이아웃/활성 뷰어 수를 줄이면 대량 적용 실수를 줄일 수 있다.

## 결과
정성 효과:
1. 반복 필터 설정 시간을 크게 줄인다.
2. 뷰어 간 실험 조건 일관성이 올라간다.

관찰 포인트(측정 후보):
1. N개 뷰어 설정 완료 시간(수동 vs sync-capture)
2. 파라미터 오설정 발생 건수
3. 모드별 적용 실패율/재작업률

## 강점
1. 다중 뷰어 실험에서 재현성 확보에 유리하다.
2. 모드별 내부 구조 차이를 사용자에게 노출하지 않는다.
3. 파라미터 null/삭제 케이스도 일관되게 처리한다.
4. 향후 배치 실험 자동화 기능으로 확장하기 좋다.

## 특허 문서 전환을 위한 작성 포인트
권리화 핵심 구성요소:
1. 기준 뷰어 선택 이벤트
2. 모드별 키 구조 인지 전파 로직
3. 활성 뷰어 범위 기반 일괄 반영

차별 주장 포인트:
1. 단순 복사 기능이 아닌 모드 구조 추상화 기반 동기 전파
2. 파라미터 존재/부재를 포함한 상태 일관성 처리

실험/증빙으로 보강할 요소:
1. 뷰어 수 증가에 따른 시간 절감률
2. 실수율 감소(오입력/누락) 통계

## 작성 시 주의사항/한계
1. 현재 동기화는 "현재 활성 모드" 기준이므로 범위를 과장하면 안 된다.
2. 필터 효과 자체의 우수성과 동기화 메커니즘은 분리해 설명해야 한다.
3. 특허화 시 "모드 인지형 전파 단계"를 반드시 명시해야 차별성이 살아난다.
