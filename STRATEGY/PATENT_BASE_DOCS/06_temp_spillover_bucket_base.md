# 06. 파일명 충돌 회피 TEMP 다중 버킷 스필오버 배치 (바탕 문서)

> 고지: 본 문서는 기술/전략 참고자료이며 법률 자문이 아닙니다.  
> 기준 문서: [PATENT_PORTFOLIO_ANALYSIS_KR_PCT.md](../PATENT_PORTFOLIO_ANALYSIS_KR_PCT.md)

## 문서 목적
대량 드롭 시 파일명 충돌을 자동 분산 처리하는 TEMP 스필오버 로직을 설명용으로 정리한다.  
핵심은 "충돌을 덮어쓰기로 처리하지 않고 버킷 분배로 해결"하는 점이다.

## 기능 정의
이 기능은 드롭된 이미지 파일명을 검사해 기존/대기 데이터와 충돌하면 `TEMP`, `TEMP_2`, `TEMP_3` 순으로 충돌 없는 버킷에 자동 배치한다.  
배치 과정에서 alias->key 매핑과 예약 키 집합을 사용해 비동기 상태 갱신 충돌을 줄인다.

핵심 포인트:
1. alias-key 로컬 매핑 및 슬롯 예약
2. 파일 단위 충돌 검사(기존 파일 + 대기 파일)
3. 버킷별 병합 저장 및 요약 피드백

## 사용방법
대표 시나리오:
1. 여러 이미지 파일(동일 파일명 일부 포함)을 워크스페이스로 드롭한다.
2. 시스템이 `TEMP` 슬롯부터 순차적으로 배치한다.
3. 충돌이 있으면 `TEMP_2`, `TEMP_3`로 자동 스필오버한다.
4. 완료 후 토스트 요약으로 버킷별 배치 수를 확인한다.

운영 팁:
1. 대량 배치 테스트는 동일 이름 파일을 일부 포함해 충돌 경로를 검증한다.
2. 분석 모드에서는 첫 이미지가 어떤 소스 alias로 설정되는지 함께 확인한다.

## 결과
정성 효과:
1. 대량 드롭에서 파일 덮어쓰기/누락 위험을 줄인다.
2. 사용자 개입 없이 충돌을 자동 정리해 입력 성공률을 높인다.

관찰 포인트(측정 후보):
1. 충돌 파일 처리 성공률(누락/덮어쓰기 0건 목표)
2. 총 배치 시간(파일 수 대비)
3. 충돌 발생 시 사용자 재작업 횟수

## 강점
1. 입력 단계에서 데이터 손실 리스크를 줄인다.
2. 충돌 처리 결과가 버킷별로 가시화되어 검증이 쉽다.
3. 모드별 유사 로직으로 일관된 사용자 경험을 제공한다.
4. 비동기 업데이트 상황에서도 로컬 예약 구조로 안정성을 확보한다.

## 특허 문서 전환을 위한 작성 포인트
권리화 핵심 구성요소:
1. 순차 인덱스 TEMP 버킷 생성/선택 규칙
2. 기존+대기 큐 동시 충돌 판정
3. 예약 맵 기반 비동기 충돌 회피

차별 주장 포인트:
1. 단순 rename 또는 overwrite가 아닌 워크스페이스형 분산 배치
2. 충돌 처리 결과를 사용자 검증 가능 형태로 제공

실험/증빙으로 보강할 요소:
1. 대량 드롭 충돌 시나리오별 성공률 데이터
2. 동시 드롭/반복 드롭 안정성 테스트 로그

## 작성 시 주의사항/한계
1. 범용 파일관리 특허와 충돌 가능성이 있어 "이미지 워크스페이스 맥락"을 유지해야 한다.
2. 버킷 네이밍 규칙은 구현 선택사항이므로 본질은 충돌 회피 절차에 둔다.
3. 특허 문서로 전환 시 단순 UX 설명보다 충돌 판정/배치 단계 순서를 명확히 써야 한다.
